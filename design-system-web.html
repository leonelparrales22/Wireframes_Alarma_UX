Reglas de sincronizacion DS Mockup

Criterios para mantener el Design System alineado con cambios de UI sin perder consistencia ni introducir componentes no definidos.

1) Si cambia una clase visual en `mockup-web.html`, registrar token o patron equivalente en este documento.
2) No agregar nuevas pantallas o componentes en mockup sin primero listarlos en "Analisis de Pantallas" y "Estados y Variaciones".
3) Mantener el login hero como caso especial del sistema web (pantalla de entrada), sin alterar el flujo funcional base.
12. Recursos Visuales Finales Utilizados

Fondos y atmosfera

Fondo claro (`#FFFFFF -> #F3F4F6 -> #EEF2F7`) con manchas radiales suaves sky/success para profundidad sin oscurecer la interfaz.

Texturas

Textura abstracta generada en CSS con `repeating-linear-gradient` (sin imagen externa) para evitar superficies planas.

Tipografias

`Space Grotesk` (titulos/KPI) y `IBM Plex Sans` (cuerpo/controles), mismas familias que mobile para continuidad de producto.

Iconografia

Iconos SVG inline para menu, estados y acciones. No se usaron sets de terceros en esta entrega.

Graficas y data-viz

Area chart, barras, lineas y mini-sparkline construidas con CSS/SVG propios; sin librerias externas de charts.

Fotografias / imagenes raster

No se utilizaron fotografias ni texturas raster externas. La interfaz se resolvio con recursos vectoriales y gradientes nativos.

Sombras y profundidad

Sistema de sombras suaves multicapa para separar niveles: `panel`, `card` y `modal`.

Comentarios colaborativos

Componente reutilizado `CommentsComponent.js` conectado a Supabase (`comentarios_entregas`) y anclado por `entrega_id` unico para esta version de mockup web.
